# Scrapyards.io

A Next.js 15 (App Router) directory and price reference site for scrap metal yards across the United States. Stack: pnpm workspaces, Node.js 24, TypeScript 5.9, Next.js 15 + React 19, PostgreSQL + Drizzle ORM, Zod validation. Front-end uses ISR; API routes are Next.js Route Handlers. Migrated from a WordPress GeoDirectory dump on 2026-05-09 — currently 51 states, 3,493 cities, 7,722 active yards, 9 metal categories, 22 metals.

## Run & operate

- `pnpm --filter @workspace/scrapyards run dev` — run the web app (workflow-managed)
- `pnpm --filter @workspace/api-server run dev` — run the API server (port 5000)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/scripts run seed-scrapyards` — seed states, cities, metals, prices, and sample yards

**Env:**
- `DATABASE_URL` (required) — Postgres connection string
- `CRON_KEY` (optional) — secret for `/api/cron/update-prices`
- `NEXT_PUBLIC_MAPBOX_TOKEN` — Mapbox public token (URL-allowlist scoped pre-launch — see `docs/maps.md`)
- `SESSION_SECRET` — managed via Replit secrets

**Deploy flow:** Replit publish (Next.js standalone build). Pre-cutover steps: lock Mapbox token URL allowlist; replace `hello@scrapyards.io` placeholder if a different inbox is preferred; confirm GeoNames CC BY 4.0 credit line in footer.

## Architecture decisions

- Next.js App Router with ISR (revalidate: 900s for prices, 3600s for yards) for SEO + performance
- Scrapyards app imports `@workspace/db` directly (server components only — no API layer needed)
- Legacy URL redirects: static `next.config.ts` + dynamic DB-backed middleware fallback
- Price cron blends national spot prices with user-reported yard prices (weighted by recency and volume)
- Schema-level JSON-LD (LocalBusiness, UnitPriceSpecification, FAQPage, etc.) on every yard, price, and item page
- All long-form prose (yard, city, category descriptions) generated by deterministic slot-filling templates — no LLM, $0 API cost, validator-gated. See `docs/content-templating.md`.
- Singleton-city directory pages (1 active yard, 2,181 of 3,493 cities) emit `noindex, follow` and are excluded from sitemap to avoid AdSense thin-content flags

## Gotchas

- Metal slugs in the DB are granular (`bare-bright-copper`, `copper-1`, `aluminum-cans`, etc.) — do NOT use generic "copper" or "aluminum" as slugs in links
- The DB `@workspace/db` package exports all tables from root: `import { yardsTable } from "@workspace/db"`
- `pg` must be in `serverExternalPackages` in `next.config.ts` (native Node module, can't be bundled)
- After schema changes: `pnpm --filter @workspace/db run push`, then re-seed if needed
- Calculator API routes live under `/calculator/api/*` (NOT `/api/*` — that path is owned by the api-server artifact via the shared proxy)
